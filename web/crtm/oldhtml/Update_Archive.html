CRTM Update Archive

Jun. 29, 2005 Update

Note: This is an incremental update. Not all dependent codes have been updated.

  • User Interface: Surface structure update.
    • Removed the Type component. The coverage fractions are now used in application code to determine the gross surface type.
    • The Binary I/O module has been modified according to the Surface structure change, but the file format remains the same.
  • User Interface: GeometryInfo structure update.
    • Added more angle components as well as components that are derived (e.g. degrees to radiances conversion, secant calculation) from the user specified angle components.
    • The function CRTM_Compute_GeometryInfo has been added to the CRTM_GeometryInfo_Define module to compute the angle-derived components.
  • The CRTM_Parameters module was updated to reflect changes in the GeometryInfo structure.
  • Developer Interface: SfcOptics structure and application module update.
    • Added the Surface_Temperature component to the SfcOptics structure.
    • Added subroutines to the CRTM_SfcOptics module to compute the weighted average surface temperature from the various surface types and their coverage fraction.
    • Updated the SfcOptics application functions with calls to wrapper functions for sensor (microwave and infrared) and surface type (land, water, snow, and ice) specific algorithms. Each set of functions (forward, tangent-linear, and adjoint) is containined within its own module.

Apr. 25, 2005 Update

  • Added "Test Data Files" section. Simple data files provided to test the CRTM.

Apr. 05, 2005 Update

  • Updated test code tarballs to include updates to SpcCoeff structure definition and I/O modules.
  • SpcCoeff datafiles have all been updated to Release5.

Mar. 28, 2005 Update

  • Atmosphere structure update.
    • Extended level pressure array to include the "top" level. Previously the array was dimensioned (1:n_Layers); now it is dimension (0:n_Layers).
    • Added level temperature entries to the structure.
  • Fixed problem with out-of-date software links.
  • Updated Atmosphere I/O software. The Cloud and Aerosol data are now read/written using their own module functions to make it easier to splice in cloud and aerosol data for test data sets.

Mar. 22, 2005 Update

We have noticed an inconsistency between the web page text and the posted software in the Source Code page. The text information on this webpage is current, but the test software tarballs were not updated. Until this problem is rectified, you download the software at your own risk. The problem should be fixed by Mar. 25, 2005.

Feb. 28, 2005 Update

  • User Interface
    • Aerosol structure definition routines created for user input of aerosol profile data, with the Aerosol structure component added to the Atmosphere structure definition and manipulation routines.
    • Aerosol data I/O added to CRTM_Atmosphere_Binary_IO module functions.
  • Developer Interface
    • Replaced separate definition of aerosol scattering structure with the AtmScatter structure. Aerosol scattering code now uses the same structure as cloud scattering code.
    • Updated all calls to RTSolution functions to reflect use of AtmScatter structure for both aerosol and cloud scatterng.
    • Added Delta_Truncation component to the AtmScatter structure definition.
